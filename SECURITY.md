# Security policy

## Supported versions

Security updates are provided for the latest released minor version.

## Reporting a vulnerability

Do not open a regular issue for a suspected vulnerability. Repository administrators should create a draft advisory under **Security > Advisories**; other authorized collaborators should contact an administrator through a trusted private channel. If the repository is made public later, GitHub private vulnerability reporting should be enabled before accepting external reports.

Include the affected version, reproduction steps, impact, and any proposed remediation. Please allow a reasonable remediation window before public disclosure.

Never include real customer data, production credentials, or active license keys in a report.
